# Introduction
This project uses Unity's ML-Agents toolkit to train intelligent Agents.
# Explanation of folder structure
## .vscode
Contains configuration files for [Visual Studio Code](https://code.visualstudio.com/). Auto-generated.
## Assets
Contains all resources such as models, scripts, prefabs, etc.
### Assets / Images
Contains image files.
### Assets / ML-Agents
Contains details about agent training using [ML-Agents (Machine Learning Agents)](https://github.com/Unity-Technologies/ml-agents). Auto-generated.
### Assets / Materials
Contains [material components](https://docs.unity3d.com/Manual/materials-introduction.html) that define the visual appearance of objects in the game **(.mat files)**.
### Assets / NNModels
Contains [ONNX models (Open Neural Network Exchange)](https://onnx.ai/), which are trained neural network models **(.onnx files)**.
### Assets / Prefabs
Contains preconfigured [GameObjects](https://docs.unity3d.com/Manual/class-GameObject.html) that can be reused such as agents, props and environments **(.prefab files)**.
### Assets / Scenes
Contains [Unity scenes](https://docs.unity3d.com/Manual/CreatingScenes.html)  **(.unity files)**.
### Assets / Scripts
Contains all C#-scripts for agents, UI and utility **(.cs files)**.
### Assets / TextMeshPro
Contains assets of the [TextMeshPro toolkit](https://docs.unity3d.com/Packages/com.unity.ugui@2.0/manual/TextMeshPro/index.html), which is used in UI design.
## Packages
Contains all external libraries and plugins used in the project. Auto-generated.
## ProjectSettings
Contains a variety of essential configuration files, such as physics configuration or input configuration. Auto-generated.
# Understanding the code
Most files can only be properly read by the [Unity Engine](https://unity.com/). An exception to this are scripts **(.cs files)**. In order to view the Project, please download the Unity Engine.
